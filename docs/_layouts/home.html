What it does

Raxicore Editor opens the container formats an engine-derived client ships next to its executable — archives, meshes, animations, textures, surfaces, and ASCII databases — and renders, inspects, and edits them through a native Vulkan-backed 3D viewport.

Asset browser

Mount a folder or open an archive straight into a role-grouped catalog — Continents, Models, Animations, Textures, Surfaces, Databases, Audio — alongside the raw on-disk tree. Archive contents load lazily.

Offscreen 3D viewport

A Silk.NET Vulkan renderer draws fully offscreen — no swapchain — and is composited into the Avalonia UI via a WriteableBitmap, sidestepping windowed-surface and device-loss crashes entirely.

Textured models

Every material resolves to its DDS texture automatically, with a per-material texture picker and a one-click empire skin swap so vehicles aren't stuck on a single livery.

Animation playback

Plays both skinned and rigid mesh-on-bone animations. The clip database auto-loads from the asset folder and is filtered to the clips that actually drive the selected model.

Full continents

Opens a continent container as placed, textured terrain plus its static scene objects — bases, towers, trees, warpgates — resolved from the map manifest and the shared object library.

Format tools

DDS/image preview, a paintable surface-type editor, structured ASCII-database table views, live-validated text editing, a hex fallback, and OBJ export.

Status

A functional viewer/editor — browsing, 3D rendering, texture selection, animation playback, and continent assembly all work today. Per-format re-pack (write-back) beyond the surface editor and OBJ export is the in-progress edge. See the supported-formats reference for exact per-format read/write status.

You need your own copy of the engine-derived game assets. No game data is included or distributed with this project or this site.
